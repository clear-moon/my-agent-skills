# 变更日志生成器

此功能将技术性的git提交转换为经过打磨、对用户友好的变更日志，您的客户和用户将能够真正理解并欣赏这些内容。

## 何时使用此功能

- 为新版本准备发布说明
- 创建每周或每月的产品更新摘要
- 为客户记录变更内容
- 为应用商店提交编写变更日志条目
- 生成更新通知
- 创建内部发布文档
- 维护公开的变更日志/产品更新页面

## 此功能能做什么

1. **扫描Git历史**：分析特定时间段或版本之间的提交
2. **对变更进行分类**：将提交归入合理的类别（功能、改进、错误修复、重大变更、安全性）
3. **从技术语言转换为用户友好语言**：将开发者的提交内容转换为客户能理解的语言
4. **专业格式化**：创建清晰、结构化的变更日志条目
5. **过滤无关信息**：排除内部提交（重构、测试等）
6. **遵循最佳实践**：应用变更日志指南和您的品牌风格

## 如何使用

### 基本用法

在您的项目仓库中：

```
从上次发布以来的提交创建变更日志
```

```
生成过去一周所有提交的变更日志
```

```
为2.5.0版本创建发布说明
```

### 指定日期范围

```
创建3月1日至3月15日期间所有提交的变更日志
```

### 自定义指南

```
根据v2.4.0以来的提交创建变更日志，并使用CHANGELOG_STYLE.md中的变更日志指南
```

## 示例

**用户**：“创建过去7天提交的变更日志”

**输出**：

```markdown
# 更新 - 2024年3月10日那周

## ✨ 新功能

- **团队工作区**：为不同项目创建单独的工作区。邀请团队成员，让一切井井有条。

- **键盘快捷键**：按?查看所有可用快捷键。无需使用鼠标，更快地导航。

## 🔧 改进

- **更快同步**：文件在设备间的同步速度提升2倍
- **更好的搜索**：搜索现在包含文件内容，而不仅仅是标题

## 🐛 修复

- 修复了大图片无法上传的问题
- 解决了定时发布中的时区混乱问题
- 修正了通知徽章计数
```

**灵感来源**：来自Lenny通讯中Manik Aggarwal的使用案例

## 提示

- 从git仓库根目录运行
- 指定日期范围以获取针对性的变更日志
- 使用您的CHANGELOG_STYLE.md以保持格式一致
- 发布前查看并调整生成的变更日志
- 将输出直接保存到CHANGELOG.md
